# Hi, I'm Bill.

I build systems — *software, simulations, and games* — that stay understandable even when the rules get messy.

# Background
My career started when I was talent-scouted out of a CS program by a land resource management firm, where I spent twenty years building production software for the timber industry until my department was laid off. The most significant thing I worked on was a long-range ecological simulation engine: a system that modeled tree growth, harvesting, and replanting cycles across millions of acres of real land, projecting outcomes in five- and ten-year increments over a hundred-year horizon. It was rigorous, domain-specific, and commercially significant enough that a subsequent team rebuilt and now sells it as a SaaS product.

That work gave me a deep foundation in statistical modeling, spatial analysis, relational databases, and the particular discipline of writing software that people depend on.

# What I've Done Since
The platforms and tools may change, but the core problem doesn’t: building software for real users, under constraints that actually matter. That kind of work changes how you approach problems — you stop guessing and start accounting for consequences.

I've stayed current through contract work, independent projects, and formal study. I spent nine months at a game studio, where I trained in Unity and contributed to an RPG in development before the studio closed. I've been making and sharing games of my own design since 2020. I balance my project work with household management and caregiving.

I'm currently completing an Intermediate Data Analytics certification program, picking up tools like R, Tableau, Power BI, and modern SQL to complement the kind of work I’ve been doing for years.
